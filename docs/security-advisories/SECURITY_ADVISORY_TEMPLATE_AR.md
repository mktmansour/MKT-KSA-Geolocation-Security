# قالب تنبيه أمني (عربي)

استخدم هذا القالب عند نشر GitHub Security Advisory وتجهيز إصدار أمني تصحيحي على crates.io.

## الملخص

- المشروع: MKT_KSA_Geolocation_Security
- معرف التنبيه: GHSA-XXXX-XXXX-XXXX
- الشدة: عالية
- الإصدارات المتأثرة: <= 2.0.1
- الإصدار المصحح: 2.0.2
- المنصة: Rust / crates.io

## الوصف

تم رصد مشكلة أمنية في سلسلة التبعيات و/أو في سلوك أمني تشغيلي. هذا التنبيه يوضح التأثير، الإصدارات المتأثرة، وخطوات التحديث المطلوبة.

## الأثر

- نوع الخطر: ثغرة في التبعيات و/أو ضعف في ضوابط الأمان.
- الأثر المحتمل: انخفاض مستوى الثقة الأمنية في بيئات الإنتاج.
- قابلية الاستغلال: تعتمد على سياق التشغيل وطريقة التكامل.

## الإصدارات المتأثرة

- المتأثر: <= 2.0.1
- المصحح: 2.0.2

## المعالجة

1. التحديث فورًا إلى 2.0.2.
2. إعادة التحقق الصارم:
   - cargo fmt --all -- --check
   - cargo clippy --all-targets --all-features -- -D warnings
   - cargo test --all
   - cargo audit
3. تدوير الأسرار المتأثرة إذا وُجد احتمال تعرض تشغيلي.

## بيان أمني للمطورين

يلزم على المطورين ومالكي الحزمة تحديث التبعيات بشكل استباقي لتقليل التعرض للثغرات.

الحد الأدنى التشغيلي:

- متابعة التنبيهات يوميًا (Dependabot وRustSec وGitHub Security).
- دمج تحديثات الأمان مباشرة بعد نجاح CI.
- إصدار Patch أمني فور ظهور ثغرات عالية/حرجة.

## إجراءات منصة crates.io

1. رفع الإصدار المصحح (2.0.2).
2. تنفيذ Yank للإصدار المتأثر عند الحاجة حسب مستوى الخطر:
   - cargo yank --vers 2.0.1
3. مزامنة changelog ووثائق الأمن مع الإصدار الجديد.
