# 02. المعمارية

تشرح هذه الصفحة بنية المشروع من المحركات الأساسية حتى طبقة API.

![المشهد العام للنظام](../assets/system-context.svg)

## 1. طبقات المعمارية

- طبقة الواجهة: مسارات HTTP داخل src/api.
- طبقة البوابة الأمنية: API key و JWT وتقييم المخاطر وتحديد المعدل.
- طبقة الذكاء الأساسي: geo و behavior و device و network و sensors و weather.
- طبقة البيانات: SQLite مع الترحيلات داخل src/db.
- طبقة الأدوات والرصد: logger و helpers و cache و precision.

## 2. دورة حياة الطلب

![دورة الطلب](../assets/request-lifecycle.svg)

1. يدخل الطلب إلى مسار Actix Web.
2. يمر عبر التحقق الأولي وتتبع الطلب.
3. يتم التحقق من API key و JWT.
4. تطبق محركات المخاطر والسياسات الأمنية.
5. يتم الحفظ عند الحاجة في SQLite.
6. تعاد الاستجابة مع غلاف ثابت وسياق تتبع.

## 3. تركيب إشارات الثقة

![تدفق الثقة والبيانات](../assets/data-trust-flow.svg)

مجالات الإشارات:

- اتساق الموقع الجغرافي.
- شذوذ السلوك.
- ثقة بصمة الجهاز.
- مخاطر الإخفاء الشبكي.
- سلامة إشارات الحساسات.
- اتساق سياق الطقس.

## 4. المكونات التشغيلية الرئيسية

- تهيئة التطبيق: src/main.rs و src/app_state.rs.
- التسجيل والتفويض المركزي: src/api/mod.rs.
- محركات النواة: src/core/*.rs.
- وحدات الأمن: src/security/*.rs.
- قاعدة البيانات والترحيلات: src/db/*.rs و src/db/migrations.

## 5. طوبولوجيا النشر

![طوبولوجيا النشر](../assets/deployment-topology.svg)

- العملاء الخارجيون يستدعون API.
- خدمة API تطبق الحماية وتحسب قرار الثقة.
- SQLite تحفظ بيانات التشغيل والتنبيهات.
- السجلات تدعم التدقيق والاستجابة للحوادث.

## 6. الخطوة التالية

انتقل إلى [03. التعمق الأمني](03-التعمق-الأمني.md).

## كلمات بحث

معمارية أمان Rust، تصميم محرك ثقة جغرافي، نموذج مخاطر متعدد الإشارات، بوابة API آمنة.
