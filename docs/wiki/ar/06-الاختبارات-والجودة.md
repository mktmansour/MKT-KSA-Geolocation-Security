# 06. الاختبارات والجودة

تعرف هذه الصفحة بوابات الجودة واستراتيجية التحقق قبل الإصدارات.

## 1. فلسفة الجودة

- فحوص بناء ولينت قابلة للتكرار.
- اختبارات تكامل كمحور أساسي للمسارات الأمنية.
- ثقة إصدار مبنية على بوابات حتمية.

## 2. خط التحقق المحلي

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all
```

## 3. مجالات التغطية

- سلوك المصادقة والتفويض.
- عقود المنع وتحديد المعدل.
- انتشار request id عبر الاستجابات.
- اختبارات سطح API الأمني.
- سلوك قاعدة البيانات والترحيلات.

## 4. تغطية CI

الوضع الحالي يتضمن:

- بناء Rust والتحقق من التنسيق.
- فحوص Clippy بجودة صارمة.
- تحليل CodeQL.
- بوابات أمان متعددة الأدوات.

## 5. قائمة جاهزية الإصدار

- نجاح كل أوامر الجودة المحلية.
- نجاح فحوص CI على main.
- تحديث changelog بفروقات الإصدار.
- تحديث README والويكي مع أي تغيير سلوكي.
- تنفيذ dry-run للنشر قبل الرفع النهائي.

## 6. استراتيجية منع الانحدار

- إضافة اختبار تكامل لكل تغيير أمني.
- الحفاظ على ثبات error codes.
- التحقق من عدم كشف مسارات غير مقصودة.
- الحفاظ على اتساق سلوك strict بعد أي تعديل.

## 7. الخطوة التالية

انتقل إلى [07. الأسئلة الشائعة وحل المشاكل](07-الأسئلة-الشائعة-وحل-المشاكل.md).

## كلمات بحث

استراتيجية اختبار Rust، اختبارات تكامل أمني، بوابات جودة للحزم، جاهزية الإصدار.
