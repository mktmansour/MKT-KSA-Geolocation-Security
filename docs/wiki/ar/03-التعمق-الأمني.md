# 03. التعمق الأمني

توثق هذه الصفحة نموذج الأمان ونقاط الضبط وسلوك الدفاع في المشروع.

## 1. الأهداف الأمنية

- منع الوصول غير المصرح بشكل حتمي.
- كشف السلوك عالي المخاطر والحمولات المشبوهة.
- حماية توفر الخدمة عبر ضوابط متدرجة.
- الحفاظ على قابلية التدقيق واستمرارية التتبع.

## 2. سلسلة الضبط الأمني

1. تطبيع الإدخال والتحقق الأولي.
2. بوابة API key والتحقق JWT.
3. تقييم مخاطر تكيفي وتطبيق السياسات.
4. تحديد معدل مع إرشادات إعادة المحاولة.
5. استجابات أمنية منظمة وسجلات موحدة.

## 3. ملخص نموذج التهديد

| التهديد | الضبط الأساسي | الضبط الثانوي |
|---|---|---|
| وصول غير مصرح | API key + JWT | سجلات تتبع request id |
| إساءة استخدام الاعتماد | تحديد المعدل | تتبع لكل IP |
| حمولات خبيثة | تقييم مخاطر وسياسات | عقود منع ثابتة |
| ضغط على الخدمة | ضبط المهلات والسعة | حدود الاتصالات |
| انحراف سلامة البيانات | ترحيلات محكومة | اختبارات تكامل |

## 4. الملفات الأمنية

يدعم المشروع ملفات تشغيل أمنية صارمة بإعدادات متحفظة للإنتاج.

الخط الموصى به للإنتاج:

- SECURITY_PROFILE=strict
- تفعيل ضوابط المهلات والاتصالات بدقة.
- تشغيل بوابات CI قبل أي إصدار.

## 5. جاهزية الاستجابة للحوادث

- تفعيل انتشار request id من البداية للنهاية.
- تجميع السجلات الأمنية في منصة بحث موحدة.
- تشغيل حزمة التحقق قبل كل إصدار.
- توثيق أي تغيير أمني في changelog وملفات الأمان.

## 6. أوامر التحقق الأمني

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all
```
